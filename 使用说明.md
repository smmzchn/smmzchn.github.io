# 文章发布功能使用说明

## 🎯 功能概述

已为你的博客添加了完整的文章发布管理系统！现在你可以：

1. ✅ **密码保护** - 只有知道密码的人才能访问管理后台
2. ✅ **Markdown编辑器** - 带实时预览的文章编辑器
3. ✅ **直接发布到GitHub** - 一键发布，GitHub Pages自动部署
4. ✅ **权限控制** - 只有你（或你授权的人）可以发布文章

## 📁 已创建/修改的文件

1. **admin.html** (新建) - 文章管理后台
2. **posts.json** (新建) - 文章索引文件
3. **script.js** (修改) - 改为动态加载文章列表

## 🚀 使用方法

### 1. 首次使用 - 设置密码

1. 打开 `admin.html`
2. 输入你想设置的管理密码（至少4位）
3. 点击"登录" - 这会设置你的管理密码
4. 下次登录需要使用这个密码

### 2. 配置GitHub发布

在管理后台的"GitHub 发布配置"部分填写：

- **GitHub Personal Access Token** - 需要`repo`权限的Token
  - 获取方法：GitHub → Settings → Developer settings → Personal access tokens → Generate new token
  - 勾选 `repo` 权限
- **仓库名** - 格式：`用户名/仓库名`（如：`shili/shili-blog`）
- **分支名** - 一般是 `main` 或 `master`

填写后点击"保存配置"

### 3. 发布新文章

1. 点击"新建文章"
2. 填写文章标题、日期、分类、摘要
3. 在内容框写文章正文（支持Markdown格式）
4. 点击"预览"可以查看效果
5. 完成后点击"发布到 GitHub"

文章会自动：
- 创建 `posts/文章id.md` 文件
- 更新 `posts.json` 文章列表
- GitHub Pages 会自动重新部署你的网站

### 4. 管理现有文章

- 在文章列表中可以看到所有文章
- 点击"编辑"可以修改文章
- 点击"删除"可以删除文章

## 🔐 权限管理

**只有知道管理密码的人才能发布文章。**

如果你想授权其他人发布文章：
1. 把管理密码告诉他们，他们就能登录admin.html
2. 或者，你可以帮他们发布（他们把文章发给你，你用admin.html发布）

**更安全的方案（推荐）**：
- 不要把GitHub Token告诉别人
- 只分享管理密码，这样他们可以在你的电脑上发布文章
- 或者，你审核后发布他们的文章

## ⚠️ 注意事项

1. **备份管理密码** - 如果忘记密码，需要清除浏览器的localStorage
2. **GitHub Token安全** - 不要把Token公开或分享给他人
3. **文章ID** - 自动从标题生成，用于文章URL（如：`我的文章` → `我的文章`）
4. **首次发布** - 确保GitHub仓库已启用GitHub Pages

## 🐛 故障排除

**问题：发布失败**
- 检查GitHub Token是否有`repo`权限
- 检查仓库名是否正确
- 检查分支名是否正确（一般是`main`）

**问题：文章发布后网站不显示**
- 检查GitHub Pages是否正在部署（在仓库的Actions标签页查看）
- 检查posts.json是否正确更新
- 清除浏览器缓存重试

**问题：忘记管理密码**
- 在浏览器控制台执行：`localStorage.removeItem('shili_blog_admin_password')`
- 刷新页面，重新设置密码

## 📧 需要帮助？

如果遇到问题，可以：
1. 检查浏览器控制台（F12）查看错误信息
2. 确认GitHub Token权限正确
3. 确保仓库已启用GitHub Pages
